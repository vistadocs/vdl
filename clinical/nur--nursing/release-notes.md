> Department of Veterans Affairs Technical Service

> Clinical Applications Product Line

> Release Notes

> The following is a summary of software enhancements provided in Nursing V. 4.0:

1.  Administration Module:
    1.  File and Administrative Site File Function changes:
        1.  FTEE SERVICE BUDGET (1106B) FILE, EDIT \[NURAFL-BFU\]:

> Previously there was a program check to add the totals between fields 6 and 20 and if it equaled Field 1 ((01) Budgeted RN) then the user was exited from the option. If fields 6 to 20 did not equal the number of Budgeted RNs, then the user was prompted to correct the budgeted entries.

> The option has been modified so that if the sum of fields 6 through 20 does not equal field 1, the user has the option of having the computer enter the correct total into the

1.  Budgeted RN field, or interactively correcting the data in fields 6 through 20.

> After the check is performed and before exiting the user is prompted "Do you want to enter Budgeted FTEE for another facility?" if the site is a consolidated site.

2.  NURSING LOCATION FILE, EDIT \[NURSFL-LOC\]:

> Has been modified so that a Nursing Unit cannot be deactivated by changing Field 1.5 (Inactive Flag) to ‘INACTIVE’ until all active staff members have been transferred from the location.

2.  Acuity/Manhour related changes:
    1.  The Nursing Acuity/Separation-Activation Run has the following modifications:
1.  Addition of Field 56 (Acuity Task Number) to the NURS Parameter (#213.9) File to be used in determining if the AMIS Acuity- Separation/Activation batch job is currently running. Where the job is queued this field will contain the task number and when it is run manually it will contain "1". Otherwise it will have no value (i.e., null).
2.  Removal of the check that prevents execution of the patient classification options if the batch job has not run to completion for the current day (minus one). This check was removed because the batch job will now calculate acuity data over a range of dates starting at the last day it ran to completion to the current day (minus one).

> Note: It is important for facilities to update the list of members of the NURS- ADP mail group. This mail group receives alert messages when the Nursing Acuity Separation-Activation batch job does not run to completion.

3.  Correction of a bug in the Nursing Acuity/Separation-Activation Batch job which prevents AMIS data for future dates from being generated. Also the routine NURACUIT has been distributed to correct any future data erroneously generated by the batch job. This routine will be used by IRMS staff to delete bogus AMIS data for future dates.
    1.  Administrative Report Changes:
        1.  The following Resource Management reports have been modified to sort by facility and to generate facility total summary reports if a site is consolidated:
1.  Patient Category Totals-AMIS 1106 Reports \[NURAPR-RES-PC\]
2.  Patient Category Totals-Midnight Acuity Reports \[NURAPR-RES- MDPC\]
3.  Manhours Print \[NURAPR-RES-MANHR\]
4.  Workload Statistics Report (Current) \[NURAPR-RES-CURWKL\]
5.  FTEE Reports \[NURAPR-FTEE\]
    1.  The following Resource Management reports have been modified to sort by facility if the site is consolidated:
1.  Current Unclassified Patient Print \[NURAPR-RES-UNCLAS\]
2.  Daily Patient Classification Summary By Ward \[NURAPR-RES- WRDCLAS\]
    1.  Administration Reports (menu) options include the following modifications:
1.  Service Reports:
    1.  Sorting by Facility Name for consolidated sites.
    2.  Sorting by Product Line for product line sites.
    3.  Grand Totals for consolidated medical centers have been provided in addition to subtotals/totals at the end of Product Line and Facility reports.
    4.  A utility that allows users to print a summary report for each medical center without printing all of the data has been provided for consolidated sites.
2.  Location reports:
    1.  Modification of location prompt to "Select Nursing Unit (enter return for all units):".
    2.  Allows sorting for one specified or all locations for a facility or for all facilities if they are consolidated sites:
    3.  Provide facility totals (end of each facility report). Provide "Grand Total" for consolidated medical centers.
    4.  Allows sorting by one, many or all product lines.
3.  Above modifications under (a) and (b) apply to the following options:
    1.  Administrative options:
        1.  Academic Degree Reports \[NURAPR-ADGREE\]
        2.  Age Reports \[NURAPR-AGE\]
        3.  Certification Reports \[NURAPR-CERTF\]
        4.  Gender Reports \[NURAPR-GENDER\]
        5.  Grade Reports \[NURAPR-GRADE\]
        6.  License Reports \[NURAPR-LICENSE\]

> A sort by state issuing the license has been added.

7.  Military Reports \[NURAPR-MILITARY\]
8.  Nursing Degree Reports \[NURAPR-NDGREE\]
9.  Proficiency Reports \[NURAPR-PROF\]
10. Comb. Acad./Nurs. Degree Reports \[NURAPR-CDGREE\]
11. Salary Reports \[NURAPR-SAL\]
12. Telephone Number Listings \[NURAPR-PHONE\]
13. NPSB Reports \[NURAPR-NPSB\]
2.  Nursing Education:
    1.  Training Report by Unit/Class \[NURSE-PR-MILOC\]
    2.  3 Yr. Training Report by Unit \[NURSE-PR-MI3LC\]
    3.  Mandatory Training Deficiency by Unit/Class \[NURSE-PR- MIDEF\]
    4.  AA/Funding Requests Report (132 Col) \[NURSE-PR-AAF\]
    5.  C.E. Training Attendance Summary Report (132 Col) \[NURSE- PR-CES
1.  The following modifications have been made in the options listed below: "Select WARD” has been changed to "Select Unit”.

> The associated nursing unit look-up now displays facility name behind unit name if the site is a consolidated site.

1.  Q.I. Data, Enter Edit :
1.  AMIS 1106 Acuity Edit \[NURAAM-ACUEDT\]
2.  Manhours Edit, AMIS 1106a \[NURAMN-MANED\]
3.  Ward, Review/Edit Classifications By \[NURAPC-REVWRD\]
    1.  Patient Care Data, Enter Edit:
1.  Classify Patients by Ward \[NURAPC-WRD\]
2.  Unclassified Patients, Edit \[NURAPC-UNCWRD\]
    1.  Patient Care Data, Print:
1.  Classification Report \[NURAPP-WRDCLAS\]
2.  Patient Assignment Worksheet \[NURCPP-ASSWRK\]
3.  Ward Census, Print \[NURSPP-LOCWRD\]
4.  Current Unclassified Patient Print, by Location \[NURAPP-UNCLOC\]
    1.  Clinical Site File Functions:
1.  Ward Activation Patient Update \[NURSPT-WRDACT\]
2.  Ward Deactivation Patient Update \[NURSPT-WRDINA\]
    1.  Special Functions (menu) options:

> The following report is sorted by facility if the site is consolidated:

1.  Employee Activation/Separation Report \[NURAED-BATSEP\].
2.  The following modifications have been made in the options listed below:
    1.  Sort by facility if site is consolidated.
    2.  Nursing unit look up displays the facility name behind the name if the site is a consolidated site.
1.  Unclassified AMIS 1106 Patients \[NURAAM-UNC\]
2.  Manhours Exception Print by Service \[NURAMN-STA\]
3.  Unclassified Midnight Patients \[NURAAM-MS-UNC\]
    1.  Admit/Transfer Pt. (Nursing Pkg. Only) \[NURSPT-ACT\] option contains the following modifications:

> Prompt "NURS LOCATION:" has been changed to "Select Nursing Unit:"

> Nursing unit look-up displays the facility name behind name of nursing unit if the site is a consolidated site.

> Prompt "NURS BED SECTION" has been changed to " Select Nursing Bed Section".

2.  Clinical Module:
    1.  Nursing Vitals/Measurements reporting interface.
        1.  Linear graphic reports now include a Pulse Oximetry/Respiration Graph.
        2.  Linear graphic reports are compatible with the HP LaserJet 5 printer.
        3.  Reports have been modified to print central venous pressure, circumference and pulse oximetry measurements. The report options are:
1.  Latest Vitals Display for a Patient
2.  Latest Vitals by Location
3.  Cumulative Vitals Report
4.  Vital Signs Record
    1.  The option Expanded SF 511 Report (Itemized I/O) has been deleted from the Nursing software.
    2.  The option Latest Vitals Display for a Patient now allows the user to send this report to a printer.
